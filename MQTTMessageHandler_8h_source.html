10/**
11 * @class MQTTMessageHandler
12 * @ingroup communication
13 * @brief MessageHandler implementation for sending messages via MQTT using
14 * PubSubClient.
15 *
16 * This handler publishes TinyRobotics messages to MQTT topics using the popular
17 * Arduino PubSubClient library. Each supported message type is sent as a binary
18 * payload (the full Message<T> struct) to a topic derived from the base topic
19 * and message type.
20 *
21 * @note Dependency: PubSubClient (https://github.com/knolleary/pubsubclient)
22 * Install via Arduino Library Manager: "PubSubClient" by Nick O'Leary
23 *
24 * Usage Example:
25 * #include <PubSubClient.h>
26 * #include <TinyRobotics/communication/MQTTMessageHandler.h>
27 * WiFiClient wifiClient;
28 * PubSubClient mqttClient(wifiClient);
29 * tinyrobotics::MQTTMessageHandler handler(mqttClient, "robot/");
30 * ...
31 * handler.onMessage(msg); // Publishes msg to MQTT
32 *
33 * The MQTT client must be connected before calling onMessage().
34 * The topic for each message is baseTopic + type (e.g., "robot/value",
35 * "robot/coordinate").
36 *
37 * Supported message types:
38 * - Message<float>
39 * - Message<Coordinate<float>>
40 * - Message<GPSCoordinate>
41 * - Message<MotionState3D>
42 *
43 * The payload is the binary representation of the full Message<T> struct.
44 */
45
